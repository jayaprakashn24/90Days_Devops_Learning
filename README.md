# 90DaysOfDevOps

**Learn • Build • Practice • Become Job-Ready**

Welcome to 90DaysOfDevOps, a structured and hands-on DevOps challenge by Jayapraksh.

This repository is designed to help you build real DevOps skills step by step in 90 days—not by watching endless videos, but by consistently doing daily tasks, building projects, and developing the mindset of a production-ready DevOps engineer.

This is not a theory-heavy course.
This is a discipline and execution-focused challenge.

---

## What is 90DaysOfDevOps?

90DaysOfDevOps is a day-wise DevOps learning challenge where:

* Each day focuses on one clear and practical task
* Every task is aligned with real-world DevOps use cases
* Learners maintain a public GitHub record of their progress
* Concepts are reinforced through hands-on implementation
* Learning is supported by structured guidance and sessions

By the end of 90 days, you will have:

* Strong DevOps fundamentals
* Experience working on multiple hands-on mini-projects
* One complete end-to-end DevOps capstone project
* A GitHub profile that reflects consistency and practical experience
* Confidence to approach DevOps interviews and real production scenarios

---

## Who Is This For?

This challenge is suitable for:

* Students and freshers starting their DevOps or Cloud journey
* Working professionals transitioning into DevOps, SRE, or Cloud roles
* Developers who want to understand infrastructure and CI/CD practices
* Anyone committed to consistent learning and improvement

No prior DevOps experience is required.
However, commitment and consistency are essential.

---

## Repository Structure

Each `day-XX` folder includes:

* Task description
* Required commands or scripts
* Notes and learnings
* Screenshots or outputs where applicable

---

## How the Challenge Works

* One day is dedicated to one task
* Tasks follow a structured and progressive learning path
* Some days focus on core concepts, others on hands-on practice
* Regular commits are encouraged to track progress
* Spending 30 to 60 minutes daily is sufficient if done consistently

---

## What You Will Learn

* Linux fundamentals and troubleshooting
* Shell scripting and automation
* Networking basics for DevOps
* Git and GitHub workflows
* Docker and containerization
* AWS core and advanced services
* CI/CD using Jenkins, GitHub Actions, and GitLab
* DevSecOps fundamentals
* Kubernetes, Helm, and ArgoCD
* Infrastructure as Code using Terraform and configuration management with Ansible
* Monitoring and observability using Grafana, Prometheus, and OpenTelemetry
* Building a complete end-to-end DevOps project

---

## How to Participate

1. Fork this repository
2. Clone your forked version
3. Navigate to the relevant `day-XX` folder
4. Complete the assigned task
5. Commit and push your changes

---

## Rules to Follow

* Avoid skipping days unless absolutely necessary
* Do not copy content without understanding it
* Document your daily learnings clearly
* Focus on practical execution rather than passive learning
* Maintain clean and meaningful commit history
